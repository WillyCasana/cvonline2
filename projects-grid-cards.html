En esta sección pueden ver algunos sistemas que he publicado en la nube(cloud) mediante video o me solicitan acceso para que puedan ingresar y realizar registros de prueba
SPA - 2
  • http://sisplaspa2wcs.azurewebsites.net/

  • Descripción: 
    Sistema web responsive para el registro de actividades diarias diseñado con la arquitectura Single Page Application(SPA)

  • Tecnología usada: Highcharts, Angularjs, Bootstrap, Azure, WebApi2, C#, Ado.Net, MySql, Google Cloud Platform.

  • Pros:
     Brinda una experiencia de usuario   similar a una aplicación de escritorio.
    Bajo consumo de ancho de banda.
    Muy buen tiempo de respuesta.

    Contras:
    Para un desarrollo ágil se requiere un buen manejo del html, css y javascript(front-end).
    Para diseñarlo a manera adaptable a cualquier equipo(responsive) se requiere cierto domino del bootstrap u otro framework css.

  • Solicitar acceso : 
ASP.NET MVC
  • http://sisplamvcwcs2.azurewebsites.net/

  • Descripción: 
    Sistema web para el registro de actividades diarias diseñado con el patrón de arquitectura Model View Controller(MVC)

  • Tecnología usada: Highcharts, Razor, Bootstrap, Asp.Net MVC, Azure, C#, Ado.Net, MySql, Google Cloud Platform.

  • Pros:
    Fácil integración con frameworks js y css.
    Su estructura se asema mas a la de una web haciendo más factible realizar programación del lado del cliente(navegador) evitando asi sobrecargar al servidor.
    Contras:
    Se tarda más tiempo en desarrollarse a comparación del desarrollo con Web forms.

  • Solicitar acceso : 
WISEJ(Web Forms)
  • http://sisplawisejwcs.azurewebsites.net/

  • Descripción: 
    Sistema web para el registro de actividades diarias hecho con el framework Wisej(Web forms)

  • Tecnología usada: Highcharts, Wisej, Asp.Net Web forms, C#, Azure, Ado.Net, MySql, Google Cloud Platform.

  • Pros:
    Permite desarrollo rápido de aplicaciones(RAD) a través de los diseñadores visuales.
    La curva de aprendizaje en desarrollo web es menor para los desarrolladores de escritorio.
    Contras:
    No brinda una buena experiencia de usuario por los continuas recargas de páginas(postback) que se producen en su uso.
    Sobrecarga y uso mas intensivo de los recursos del servidor web.
    Tiempo de respuesta no tan óptimo.

  • Solicitar acceso :
